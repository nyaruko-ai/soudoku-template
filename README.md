# soudoku-template

`soudoku-template` は、Codex CLI を使って AI と対話しながら「創読(そうどく)」を楽しむためのテンプレートです。

このテンプレートの目的は、AI に小説を一気に書かせることではありません。  
また、作った小説で収益化したり、作家になることを主目的にするものでもありません。

創読の中心にあるのは、**これまで自分では物語を書けなかった人でも、AI を道具として使うことで、自分が本当に読みたいシチュエーションや関係性や展開を持つ小説を形にし、それを自分自身で楽しむこと**です。

つまりこれは、創作支援ツールであると同時に、**自分のための新しい娯楽体験**を作るためのテンプレートです。

GitHub Pages での公開や `docs/` への Web 出力は、その体験を見やすくしたり、あとから振り返ったり、人に見せたりするための手段です。  
主役はあくまで「創読すること」そのものです。

このリポジトリに含まれる `docs/` はサンプル作品です。  
本来の用途は、このテンプレートをコピーして自分の小説プロジェクトを作ることです。

## 創読(そうどく)とは

創読とは、**AI を道具として使い、自分が読みたい小説を自分で作りながら、その場で自分で楽しむ体験を表した造語**です。

ここで大事なのは、「上手に小説を書けること」ではありません。  
創読は、これまで文章を書くのが難しかった人でも、自分の頭の中にある

- 見たいシチュエーション
- 好きなキャラクター関係
- 好みの雰囲気
- 読みたい展開

を AI と一緒に形にしていけることに価値があります。

普通の小説制作では、まず書き上げてから読むことが多いですが、創読では次のように進みます。

- まずテーマやキャラクターを決める
- イメージ画像や背景画像を作る
- 章や話を少しずつ書く
- その時点の本文と画像をすぐ読める形で確認する
- 気に入ったら続きを作り、違和感があれば直す

つまり、完成後に読むのではなく、**自分のための物語を、作りながら読み、読みながら育てる**ことが創読です。

このテンプレートは、その体験を Codex CLI 上で進めやすくするためのものです。

## これは何か

このテンプレートを使うと、Codex に次のような依頼ができます。

- 小説のテーマを渡して、プロットとキャラクターを一緒に固める
- キャラクター画像、背景画像、節ごとのシーン画像を作る
- `章 > 話 > 節` 単位で段階的に本文を作る
- 完成した内容をモバイル向け Web コンテンツとして `docs/` に出力する
- 必要なら `docs/` を GitHub Pages で公開する

## 必要なもの

- Node.js 18 以上
- npm
- Codex CLI
- Gemini API キー

## 重要な注意

### Gemini API キーが必要です

画像生成には Gemini API を使います。  
そのため、利用前に `.env` に API キーを設定する必要があります。

例:

```env
GEMINI_API_KEY=your_api_key
```

### 画像生成にはコストがかかります

キャラクター画像、背景画像、節画像、タイトル画像はすべて Gemini API による画像生成です。  
そのため、**画像を生成するたびにコストが発生します**。

- 作り直すたびに追加コストが発生します
- 節数が多いほど、画像生成コストも増えます
- 利用前に Google の Gemini API 公式料金ページを確認してください

### 利用している画像生成モデル

このテンプレートで現在使っているモデルは次です。

- `gemini-3.1-flash-image-preview`

## 使い方

### 1. このリポジトリを取得する

GitHub から clone するか、ZIP でダウンロードします。

```bash
git clone https://github.com/nyaruko-ai/soudoku-template.git
cd soudoku-template
```

### 2. 依存を入れる

```bash
npm install
```

### 3. Gemini API キーを設定する

`.env.example` を参考に `.env` を作成し、`GEMINI_API_KEY` を設定します。

```bash
cp .env.example .env
```

### 4. Codex CLI でこのフォルダを開く

このリポジトリのルートで Codex CLI を起動します。

```bash
codex
```

### 5. Codex に小説制作を依頼する

Codex には、普通の自然文で依頼すれば大丈夫です。  
たとえば次のように始めます。

```text
このテンプレートを使って小説を作りたいです。
テーマは、休日を過ごす中年男性が実は重要人物で、
本人の知らないところで猫耳メイド護衛が守っている話です。
まずプロット、主要キャラクター、背景イメージを作ってください。
```

または、もっと短くても構いません。

```text
このテンプレートで創読小説を始めたいです。まず初期設計から進めてください。
```

## Codex と進めるときの基本イメージ

このテンプレートは、いきなり全文を書き切る用途ではありません。  
Codex と次の順番で段階的に進める想定です。

1. テーマ、ジャンル、ボリュームを伝える
2. プロットと主要キャラクターを固める
3. キャラクター画像と背景イメージを確認する
4. 第1章から順に本文を作る
5. 章ごとに画像と Web 表示を確認する
6. 必要なら修正する
7. 最後にタイトル画像を作る

## Codex に依頼するときの例

### 新しく始める

```text
このテンプレートで新しい創読小説を作りたいです。
テーマは未来都市の探偵ものです。
まず初期設計から始めてください。
```

### 続きを作る

```text
この小説の続きを進めてください。
第2章第1話の本文から進めて、レビュー後に画像と docs ビルドまでお願いします。
```

### 画像だけ直す

```text
第1章第2話第1節の画像だけ直してください。
主人公が一人でいるように見せたいです。
```

### タイトル画像を最後に作る

```text
全体が固まったので、現在のキャラクター画像に合わせてタイトル画像を作ってください。
```

## GitHub Pages で公開する

このテンプレートは `docs/` をそのまま GitHub Pages で公開できます。

### 手順

1. 自分の GitHub リポジトリへ push する
2. GitHub のリポジトリ画面で `Settings` を開く
3. `Pages` を開く
4. `Build and deployment` で `Deploy from a branch` を選ぶ
5. Branch に `main` を選ぶ
6. Folder に `/docs` を選ぶ
7. 保存する

しばらくすると、`docs/` の内容が GitHub Pages で公開されます。

## 含まれているサンプルについて

このリポジトリには、サンプルとして 1 作品ぶんの `docs/` が含まれています。  
これはテンプレートの最終出力イメージを確認するためのもので、テンプレート自体の本質は Codex と一緒に新しい小説を作っていくフローにあります。

## 補足

- 制作中データは `project/` に蓄積されます
- 配信用の成果物は `docs/` に出力されます
- 画像生成は必要な場面だけ実行するほうがコストを抑えられます
